Nova 文档里给出的核心接法，本质上是 **OpenAI 兼容调用**：

* base URL 用 `https://once.novai.su/v1`
* 认证方式是 `Authorization: Bearer sk-...`
* Python 示例是 `from openai import OpenAI` 后再调用 `client.chat.completions.create(...)`。([Nova AI 官方文档][1])

你现在的项目是 **FastAPI + 多 Agent**，所以建议你按下面这层来接。

---

## 一、推荐接入位置

不要这样：

```python
class WorkerAgent:
    def run(self, query):
        client = OpenAI(...)
        return client.chat.completions.create(...)
```

因为这会导致：

* 每个 Agent 都重复写一遍模型调用代码
* 后面切换到别的 API 很麻烦
* 重试、超时、日志、计费统计没法统一做

更合理的是四层：

### 1. `config`

放密钥、base_url、默认模型名

### 2. `llm/provider`

专门封装 Nova API 的调用

### 3. `agents`

每个 Agent 只管：

* system prompt
* 输入组织
* 输出解析

### 4. `orchestrator / planner`

负责任务分发，调用多个 Agent

也就是：

**Agent 不直接碰 HTTP 细节**
**Agent 只依赖一个抽象好的 LLMClient**

## 四、封装一个统一的 Nova Client

### `app/llm/nova_client.py`

```python
from typing import List, Dict, Any, Optional
from openai import OpenAI
from app.core.config import settings


class NovaClient:
    def __init__(
        self,
        api_key: Optional[str] = None,
        base_url: Optional[str] = None,
        model: Optional[str] = None,
        timeout: Optional[int] = None,
    ):
        self.api_key = api_key or settings.NOVA_API_KEY
        self.base_url = base_url or settings.NOVA_BASE_URL
        self.model = model or settings.NOVA_MODEL
        self.timeout = timeout or settings.NOVA_TIMEOUT

        self.client = OpenAI(
            api_key=self.api_key,
            base_url=self.base_url,
            timeout=self.timeout,
        )

    def chat(
        self,
        messages: List[Dict[str, str]],
        model: Optional[str] = None,
        temperature: float = 0.2,
        max_tokens: int = 2000,
    ) -> str:
        response = self.client.chat.completions.create(
            model=model or self.model,
            messages=messages,
            temperature=temperature,
            max_tokens=max_tokens,
        )
        return response.choices[0].message.content or ""

    def chat_raw(
        self,
        messages: List[Dict[str, str]],
        model: Optional[str] = None,
        temperature: float = 0.2,
        max_tokens: int = 2000,
    ) -> Any:
        return self.client.chat.completions.create(
            model=model or self.model,
            messages=messages,
            temperature=temperature,
            max_tokens=max_tokens,
        )
```

这里的写法就是沿着文档示例来的：使用 `OpenAI(api_key=..., base_url="https://once.novai.su/v1")`，再调 `client.chat.completions.create(...)`。([Nova AI 官方文档][1])

## 六、每个 Agent 只关心自己的 Prompt 和输出

比如你项目里的 Router Agent：

### `app/agents/router_agent.py`

```python
import json
from app.agents.base_agent import BaseAgent


class RouterAgent(BaseAgent):
    SYSTEM_PROMPT = """
你是一个任务路由智能体。
你的职责：
1. 判断用户问题属于哪个业务域
2. 识别应调用哪些 worker
3. 输出严格 JSON

输出格式：
{
  "domain": "...",
  "workers": ["worker_a", "worker_b"],
  "reason": "..."
}
"""

    def run(self, user_input: str):
        messages = [
            {"role": "system", "content": self.SYSTEM_PROMPT},
            {"role": "user", "content": user_input},
        ]
        content = self.llm.chat(messages, temperature=0.0, max_tokens=1000)

        try:
            return json.loads(content)
        except Exception:
            return {
                "domain": "unknown",
                "workers": [],
                "reason": f"模型输出非 JSON：{content}"
            }
```

再比如 Worker Agent：

### `app/agents/worker_agent.py`

```python
from app.agents.base_agent import BaseAgent


class WorkerAgent(BaseAgent):
    def __init__(self, llm, table_yaml: str, worker_name: str):
        super().__init__(llm)
        self.table_yaml = table_yaml
        self.worker_name = worker_name

    def run(self, user_input: str):
        system_prompt = f"""
你是表级 SQL 分析智能体：{self.worker_name}
你只能基于如下表结构进行分析：
{self.table_yaml}

请完成：
1. 识别和该表相关的查询条件
2. 给出候选字段
3. 给出 where 子句建议
4. 不要编造不存在的字段
"""

        messages = [
            {"role": "system", "content": system_prompt},
            {"role": "user", "content": user_input},
        ]

        return self.llm.chat(messages, temperature=0.1, max_tokens=1500)
```

## 八、在编排器里串起来

### `app/services/agent_orchestrator.py`

```python
from app.llm.factory import get_default_llm
from app.agents.router_agent import RouterAgent
from app.agents.worker_agent import WorkerAgent


class AgentOrchestrator:
    def __init__(self):
        self.llm = get_default_llm()
        self.router = RouterAgent(self.llm)

    def run(self, user_input: str):
        route_result = self.router.run(user_input)

        workers_output = []
        for worker_name in route_result.get("workers", []):
            table_yaml = self._load_yaml_for_worker(worker_name)
            worker = WorkerAgent(
                llm=self.llm,
                table_yaml=table_yaml,
                worker_name=worker_name
            )
            result = worker.run(user_input)
            workers_output.append({
                "worker": worker_name,
                "result": result
            })

        return {
            "route": route_result,
            "workers_output": workers_output
        }

    def _load_yaml_for_worker(self, worker_name: str) -> str:
        # 这里先写死，后面再接文件系统/数据库
        return f"tables for {worker_name}"
```

## 十、你这个项目里最该注意的 5 个点

### 1）不要把 API Key 写进代码

只放 `.env`，绝对不要提交到 git。

### 2）给 Agent 输出加结构化约束

你是多智能体系统，最怕“自由文本漂移”。
所以 Router、Planner、Top Agent 最好都输出：

* JSON
* 或 Pydantic 可解析格式

### 3）一定要做异常处理

比如：

* API 超时
* 返回空结果
* 模型输出不是 JSON
* 限流 / 配额异常

例如可以这样包一层：

```python
def safe_chat(self, messages):
    try:
        return self.chat(messages)
    except Exception as e:
        return f"LLM调用失败: {str(e)}"
```

### 4）不同 Agent 不一定用同一个模型

Nova 文档示例里展示了模型名是放在 `model` 字段里的，所以你完全可以按 Agent 类型切换模型。([Nova AI 官方文档][1])

例如：

```python
router_llm = NovaClient(model="快模型")
top_llm = NovaClient(model="强模型")
```

### 5）把“提示词”和“调用器”分开

很多人一开始把 prompt、解析、HTTP 调用全塞在一个类里，后面会非常难维护。

你最好分成：

* `prompts/`
* `agents/`
* `llm/`
* `services/`

## 十一、如果你要做“真正的 Agent 调用链”，下一步该怎么扩展

你现在先别急着一步做到最复杂，建议分三阶段。

### 阶段 1：最小可运行骨架

先跑通：

* FastAPI 接口
* 一个 NovaClient
* 一个 RouterAgent
* 一个 WorkerAgent

只要 `/api/agent/run` 能返回结果，就成功了。

### 阶段 2：结构化输出

给 Router / Worker / Top Agent 全部加：

* JSON schema
* 输出校验
* 失败重试

### 阶段 3：多 Agent 编排

再上：

* Planner
* Memory
* Tool calling
* SQL 生成与校验
* 多轮修正

这个顺序最稳。

## 十二、给你一个更贴近你项目的落地建议

结合你现在做的 **医院 Text-to-SQL / 多智能体系统**，我建议你这样映射：

* **RouterAgent**：判断问题涉及哪些业务表/子域
* **WorkerAgent**：基于各自 YAML 表结构做字段匹配、条件提取、候选 SQL 片段生成
* **TopAgent**：整合多个 Worker 输出，形成最终 SQL
* **Medical/TermAgent**：先做医学术语澄清与实体标准化
* **ValidatorAgent**：检查 SQL 语法、字段合法性、是否引用不存在字段

而 Nova API 只作为它们背后的统一 LLM 推理引擎。

也就是说，**Nova 是“模型调用底座”**，不是“业务逻辑本身”。
