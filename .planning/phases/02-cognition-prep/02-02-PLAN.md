# Plan 02: Semantic Packet Typed Models and Builder Interface

## Goal

实现 `cognition/semantic_packet.py` 的强类型骨架，让系统能以 `person_id + policy_id + optional qualification scope` 为正式入口，输出五段式任务裁剪语义包：`task`、`fields`、`relations`、`time_semantics`、`dict_excerpt`。本 plan 只做内部接口与 builder 契约，不直接读取真实数据库。

## Verification

- `cognition/semantic_packet.py` 存在且可导入
- 语义包五段结构完整且字段命名稳定
- 提供 builder 或同等入口，能从正式任务参数构造一个 `SemanticPacket`
- `python -m pytest tests/test_semantic_packet.py -q` 通过

## must_haves

- 保持五段结构，不退化成 prose-only 文本
- `dict_excerpt` 默认 excerpt-first，而不是整本字典灌输
- 支持 `person_id + policy_id + optional qualification scope`
- 通过注入 provider / resolver 实现可替换数据来源，避免与现有运行时耦合
- 不修改 `/api/debate`、`/api/debate_stream`、`DebateOrchestrator`
