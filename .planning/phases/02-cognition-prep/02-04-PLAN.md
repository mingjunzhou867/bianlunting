# Plan 04: Question-Driven Evidence Planner

## Goal

实现 `cognition/evidence_planner.py`，让系统能以 `person_id + policy_id + optional qualification scope` 为输入，结合 `SemanticPacket` 与 `QuestionTemplates` 输出 question-driven 的取证计划。每个 plan item 对应一个资格问题，明确 evidence targets、relevant fields、time semantics、priority 和 missing-evidence strategy。

## Verification

- `cognition/evidence_planner.py` 存在且可导入
- planner 输出是一问一项，而不是按表分组
- plan item 含有 trace 字段，能追溯到 qualification item 和 question template
- `python -m pytest tests/test_evidence_planner.py -q` 通过

## must_haves

- 输入正式任务参数 `person_id + policy_id + optional qualification scope`
- 依赖 `SemanticPacket` 与 `QuestionTemplates`，而不是直接生成 SQL
- 输出 question-driven plan items，含 `evidence_targets`、`relevant_fields`、`time_window_or_time_rule`、`priority`、`missing_evidence_strategy`
- 保持显式链路：`qualification item -> question template -> evidence plan item`
- 不触碰现有 `/api/debate`、`/api/debate_stream` 和 `DebateOrchestrator`
