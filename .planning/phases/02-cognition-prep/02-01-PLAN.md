# Plan 01: Dictionary Generator Skeleton and Seed Assets

## Goal

建立 Phase 2 的字典资产入口：创建 `scripts/generate_dicts.py`，冻结最小生成流程和输出契约，并以 `dicts/ADC310.json` 为锚点整理出可验证的种子级字典工件路径。这个 plan 只做 generator skeleton 和 seed asset contract，不追求全量扫库。

## Verification

- `scripts/generate_dicts.py` 存在且可导入/执行帮助信息
- 生成器提供最小稳定入口，例如 `dry-run`、`manifest` 或 `write`
- 输出契约遵循 ADC310 增强格式，包含 `source_refs`、`aliases`、`notes` 的扩展方向
- `python -m pytest tests/test_generate_dicts.py -q` 通过

## must_haves

- `generate_dicts.py` 不依赖现有前端或 debate 主链
- 生成器输入/输出接口是稳定的，即使底层数据源暂时是样例或占位
- `dicts/ADC310.json` 继续作为语义字典样式锚点，而不是被新 schema 推翻
- 至少有一组可测试的“发现字典候选 -> 生成 artifact metadata -> 写入 dicts”行为
- 测试覆盖 dry-run 或 manifest 路径，避免只能靠人工看文件验证
