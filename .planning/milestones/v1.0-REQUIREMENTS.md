# Requirements Archive: v1.0

**Archived:** 2026-03-24  
**Milestone:** v1.0  
**Core Value:** A completed debate session can be saved completely and retrieved reliably later, with enough structure to support audit, reuse, and future product features.

## Final Requirement Outcomes

### Persistence

- [x] **PERS-01**: System saves one canonical completed debate session after `/api/debate` finishes
- [x] **PERS-02**: System saves one canonical completed debate session after `/api/debate_stream` finishes
- [x] **PERS-03**: Saved session includes person identifier, session identifier, execution status, final conclusion, final stance, consensus metadata, and completion timestamps
- [x] **PERS-04**: Saved session includes the evidence snapshot used for that debate
- [x] **PERS-05**: Saved session includes round-by-round agent intermediate judgments with agent role, round number, confidence, evidence refs, reasoning, dissent points, and key finding
- [x] **PERS-06**: Database schema and runtime persistence code share one aligned contract for debate storage

### Retrieval

- [x] **READ-01**: API can fetch one saved debate session by `session_id`
- [x] **READ-02**: API can list saved debate sessions for a given `id_card`
- [x] **READ-03**: Retrieved payload is stable enough to support future frontend history pages without rerunning agents

### Compatibility

- [x] **COMP-01**: Existing `/api/debate_stream` streaming behavior continues to work for current frontend consumers
- [x] **COMP-02**: Existing `/api/debate` final response continues to work for current callers
- [x] **COMP-03**: Persistence failures are surfaced clearly in logs and code paths instead of failing silently

### Quality

- [x] **QUAL-01**: Automated tests verify debate session save behavior
- [x] **QUAL-02**: Automated tests verify saved sessions can be read back correctly
- [x] **QUAL-03**: Automated tests verify the DDL/runtime schema contract for debate persistence

### History UX

- [x] **HIST-01**: Frontend shows a left-side history list of saved debate sessions
- [x] **HIST-02**: If no history item is selected and no ID card is submitted, the center pane stays empty
- [x] **HIST-03**: Clicking one history item loads that saved debate's full process into the center pane
- [x] **HIST-04**: Entering an ID card and submitting shows the current live agent state for that person in the center pane

### Deferred Governance

- [ ] **GOV-01**: Historical debate data has access control and audit policy
- [ ] **GOV-02**: Saved session data has retention and cleanup rules
